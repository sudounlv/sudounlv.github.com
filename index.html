  • How does this whole thing work?

    • Problems will be provided every other Monday evening at 8PM PST or slightly thereafter. Generally will be in the form of a word problem, perhaps with explanatory diagrams. However, we reserve the right to change the format to be slightly more interesting.
    • Submission of your solutions should be mailed to codechallenge@sudounlv.com. Please include SUBMISSION in the subject line to make it easy on us.
    • Submission closes 48 hours after the problem is posted.
    • Resubmissions are allowed. However, the last solution one submits over the timeframe will be the one graded. If you submit an incorrect program over a correct program, or a correct program over another correct program 12 hours later, tough luck!
    • An incorrect solution is an incorrect solution. No partial credit!
    • However, we're a bit flexible about the output format, with potentially limited exceptions. You won't be disqualified if the problem states that you output "Case 0: 2" and you forget the colon, etc.
    • Until the end of the week's competition, no assertions will be provided to us as to the correctness of a program.
    • The instructions for correct submissions will be provided with the problem, but will generally follow this pattern:
      • There will be some input/output pair[s] provided with the problem that validate some level of correctness of your program.
      • There will be an input provided without its corresponding output.
      • A complete submission will involve the corresponding output for the secret input, as well as the source code.

  • But wait, can I ...

    • Use whatever language I want?
      • Yes! You definitely don't have to stick to C or C++. If the language you're using isn't among those included by default on *NIX machines (Ruby, Python, Perl, Java, C++, C, Shell, etc.), it must have a freely-available compiler/runtime and we'd really appreciate instructions on how to run it.
      • Please, no submissions in any of these languages though.
    • Use third party libraries?
      • Use of third party libraries is limited to tedium only. If the problem asks you to solve a problem like finding the minimum distance path from point X to Y over a graph, libraries that do basic string manipulation or list sorting are fine, but a 'find the min-path over this graph' library would not be OK.
      • It would be wise to stick to the standard libraries. Any non-standard libraries used should be attached to your submission in whatever format makes sense for your language.
    • Work with others?
      • Unfortunately, there is a hard balance to level between comradrie and competition. We encourage you to meet up and work on these problems, and encourage discussion and debates about the best way to solve the problem.
      • However, source code sharing is strictly verboten. If this happens, all involved parties' scores for the week will be 0.
    • Ask questions about the problem?
      • Absolutely! We encourage questions that ask about clarifications to the problem, about the competition in general, or ask about our favorite color. Unfortunately, we cannot answer questions about how best to solve the problems, whether an approach you are considering is correct, etc.
      • We'll do our best to make available answers via email (codechallenge@sudounlv.com) or in our IRC channel: #sudounlv on irc.freenode.net
      • Frequently asked questions will be posted on the code challenge page as soon as is reasonable.

  • Points!

    • Points are accumulated across the entire semester
    • Points are awarded based on the order of the correct submissions, whether it takes 30 minutes, an hour, or a day!
      • First place: 100
      • Second place: 80
      • Third place: 60
      • All others: 40
    • After the 48 hour submission period is over, our reference solution will be posted, along with the point winners for that problem.
    • At the end of the semester, we will add up the points and declare the winners!
